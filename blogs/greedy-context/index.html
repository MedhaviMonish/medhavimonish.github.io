üîó View on GitHubGreedyContext: Shrinking LLM Memory with Semantic Graphs

Traditional LLM-based chat systems send the entire conversation history to the model, even if most of it is no longer relevant. This not only increases token usage and latency but can also confuse the model during topic switches. GreedyContext solves this by constructing a semantic graph of message relationships and then extracting a minimal, meaningful subset of messages that are directly relevant to the user's latest query. So instead of blindly sending 100+ previous messages, you send only the 5‚Äì10 that matter.

‚ùå Problems with Full-History Approach

  • Includes outdated or unrelated messages just because they're part of the session
  • Treats topic switches as continuation ‚Äî when they should be fresh starts
  • Token usage grows linearly with conversation length
  • Drives up cost, latency, and risk of hallucination
  • Requires summarization layers or external memory management or custom memory modules
  • Makes it unclear which messages actually shaped the response

‚úÖ GreedyContext Advantages

  • Strips out irrelevant branches and topic switches ‚Äî treating each as a fresh intent
  • Keeps only semantically linked messages, not just recent ones
  • Reduces tokens by 50‚Äì90%, slashing cost and improving focus
  • Avoids summarization yet keeps outputs coherent
  • Less LLM confusion = fewer hallucinations and context drift
  • Plug-and-play with any LLM ‚Äî no infra or architecture change needed

üìå A Simple Example

You can pass both user and assistant messages into GreedyContext and let it trace the most relevant semantic path backward from the latest query:

python

2
3chat_messages = [
4    {"role": "user", "content": "How do I start preparing for a career in robotics?"},

6    {"role": "user", "content": "Which language is preferred‚ÄîC++ or Python?"},
7    {"role": "assistant", "content": "Python is great for prototyping, but C++ is essential for performance-critical robotics."},
8    {"role": "user", "content": "Any tips on improving productivity while studying?"},
9    {"role": "assistant", "content": "Use Pomodoro timers and remove distractions."},
10    {"role": "user", "content": "What are some good home workouts without equipment?"},
11    {"role": "assistant", "content": "Bodyweight exercises like push-ups, squats, and planks work great."},
12    {"role": "user", "content": "Can you give me a quick vegetarian recipe?"},
13    {"role": "assistant", "content": "Try stir-fried veggies with tofu and rice."},
14    {"role": "user", "content": "What is quantum entanglement in simple terms?"}
15]
16
17# Recommended: use mode='cross' for better semantic accuracy
18graph = SemanticContextGraph(chat_messages, model_name="cross-encoder/stsb-roberta-base", mode="cross")
19graph.build_graph(threshold=0.2)
20
21path = graph.greedy_path(start_node=len(chat_messages), goal_node=1)
22used_ids, relevant_messages = graph.extract_relevant_messages(path)
23
24for msg in relevant_messages:
25    print(f"{msg['role'].upper()}: {msg['content']}")

üí° Final Tip

For best results, use a threshold (e.g. 0.2) to ignore weak links. It trims unnecessary edges and lets the greedy algorithm return a cleaner and more focused message subset.
