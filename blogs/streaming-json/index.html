When generating structured responses from LLMs, streaming JSON is almost impossible. JSON parsing requires strict syntax ‚Äî every quote, comma, and bracket must be perfectly placed. But when you stream output token-by-token, the structure is often incomplete mid-way. A missing brace or an unclosed string can break the whole parser.

üí• The Problem

JSON is strict. If you're using OpenAI's streaming API and trying to render JSON in real-time, you can't do much until the full structure is delivered. That means the UI can't update until the model stops thinking ‚Äî and that's frustrating. If your JSON affects how UI is rendered then you have to wait until the end, unlike text where you can stream freely without structural constraints.

üîß The Trick: Use YAML

YAML is more forgiving. It doesn‚Äôt require commas, brackets, or perfect nesting at every step. Here's what I did:

  • Let the model output YAML instead of JSON
  • Stream it token-by-token
  • Try parsing it incrementally using a custom parser
  • Merge whatever is valid into a full JSON template

In this code I have used YamlTokenStreamer to simulate token streaming. This class is purely for local testing ‚Äî when using a real LLM with stream support (like OpenAI), you won‚Äôt need it. Just plug the rest of the logic into your streamed tokens, and you'll be able to produce fully structured JSON in real time ‚Äî with `null` values where data is not yet available.
